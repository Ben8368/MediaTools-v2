# MediaTools v2 - Refactor Plan

> **提示**：本项目采用治理文档体系，AI 工具请先阅读 `AGENTS.md`

## 🎯 重构目标

### 核心问题
Legacy 版本（https://github.com/Ben8368/MediaTools）存在以下问题：
- ❌ 包含大量 vendor 依赖（yt-dlp, capcut-mate 等）
- ❌ 大文件问题（58MB+ 配置文件无法推送）
- ❌ 密钥检测触发（vendor 代码中的示例密钥）
- ❌ 架构过度复杂（核心契约未稳定时过早前后端分离、过度抽象）

### 重构原则
1. **小巧精悍**：最小依赖，核心功能优先
2. **跨平台优先**：Windows / macOS / Linux 统一体验
3. **模块化**：功能独立，按需加载
4. **可测试**：核心逻辑与 I/O 分离
5. **可演进**：当前 CLI 优先，但 core 与适配层保持解耦，未来可增加 API / Web / 桌面入口
6. **体验延续**：前端技术栈可演进，但布局、视觉节奏和用户路径应尽量贴近 Legacy，降低用户学习成本

## 🔄 从 Legacy 迁移

### 保留价值
- 核心媒体处理逻辑
- 已验证的工作流
- 跨平台路径处理经验
- 前端布局、视觉习惯和用户操作路径

### 舍弃部分
- 所有 vendor 目录 → 使用系统工具或轻量库
- 过早完整前后端架构 → CLI 核心优先，Phase 3 以下载工作台轻前端切入
- 过度抽象层次 → 简化实现
