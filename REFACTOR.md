# MediaTools v2 - Refactor Plan

## 架构原则

1. **简单优先**：避免过度设计
2. **跨平台**：路径、编码、依赖统一处理
3. **模块化**：功能独立，按需加载
4. **可测试**：核心逻辑与 I/O 分离

## 目录结构（待定）

```
mediatools/
├── src/           # 核心代码
├── cli/           # 命令行工具
├── tests/         # 测试
└── docs/          # 文档
```

## Roadmap

- [ ] 确定技术栈
- [ ] 设计核心架构
- [ ] 实现基础模块
- [ ] 跨平台测试
- [ ] 文档完善

## 从旧版迁移

需要保留的核心功能：
- 媒体文件处理
- 自动化工作流
- 跨平台路径处理

需要精简的部分：
- 过度抽象的层次
- vendor 依赖管理
- 冗余配置
