## ⚠️ **技术决策点**

### **1. WebSocket 是否必要？**
**当前方案**：轮询（每 2 秒）  
**优点**：无需额外依赖、实现简单  
**缺点**：状态更新有延迟、浪费请求

**推荐**：Phase A/B 先用轮询，Phase C 再评估 WebSocket

---

### **2. 是否移除 AI/PS/AE 相关代码？**
**当前状态**：
- `PhotoshopApp.tsx`：82KB（856 行）
- `AIAssistantApp.tsx`：9KB
- 大量 Legacy API stub

**推荐**：
- ✅ 保留组件文件（用户可能未来需要）
- ✅ 从 `appRegistry.tsx` 暂时隐藏这些应用
- ✅ API stub 保留但添加清晰的 "v2 未实现" 注释

---

### **3. 是否需要持久化任务记录？**
**当前**：内存 `TaskStore`（重启丢失）  
**用户需求**：可能需要查看历史

**推荐分阶段**：
- Phase A：内存存储（MVP）
- Phase B：添加 SQLite 持久化（参考 Legacy `tasks.db`）
- Phase C：添加搜索和统计

## 🔍 **性能优化机会**

### **1. 前端包体积**
**当前**：353KB JS（gzipped 113KB）

**优化方向**：
- 评估是否需要所有 icon-library
- 懒加载非核心应用（PhotoshopApp 等）
- 按需导入 i18n 语言包

**预期收益**：减少 30-50KB

---

### **2. API 响应优化**
**当前**：任务列表返回完整任务对象

**优化方向**：
- 分页支持（当任务 > 100 时）
- 只返回必要字段
- 添加 ETag 缓存

**预期收益**：响应时间减少 50%（大量任务时）
